Responsive design

Now when you know basics of HTML, CSS and JS you are ready to start developing real websites and applications.

In 2020 people browse web on many different ways. Mostly on mobile, but not only. Therefore we have to make sure that our website looks great regardless of device and screen size on which it is displayed.

In the past some websites were offering 2 versions - one for desktop users, and one for mobile (and some like Facebook , are still doing this.). This of course solves our issue but it also require to have two separate version of the same page. Two teams to maintain and develop it etc.

Instead of creating two version of our web page, we can use so called Responsive Web Design in our project.

Responsive Web Design

Responsive Web Design is about using HTML and CSS to automatically resize, hide, shrink, or enlarge, a website, to make it look good on all devices (desktops, tablets, and phones):

Viewport

When making responsive web pages, add the following element in all your web pages:


    <meta name="viewport" content="width=device-width, initial-scale=1.0">

Media queries

Since RWD become a standard for modern development, also HTML/CSS evolved to make RWD development easier. Let's gave a look at media queries in CSS

Let's add following css code to our code: 


    body {
      background-color: lightcyan;
    }
    
    @media screen and (max-width: 992px) {
      body {
        background-color: lavender;
      }
    }
    
    @media screen and (max-width: 600px) {
      body {
        background-color: mistyrose;
      }
    }

Now let's resize our window

Probably you already realized. Media queries allows us to applies different styles depending on screen size.

In our particular case used values 992px and 600px. They are what we call "typical breakpoints" for devices. What does it mean?

These 3 points are considered as border point between mobile, tablet and desktop screen size.

You already know that CSS is very powerful and we can adjust almost anything on our page.

There is no surprise that in combination with media queries we can crate a totally different layouts for our page i.e.:

Replace existing styles with the following:


        * {
          box-sizing: border-box;
        }
        
        .column {
          float: left;
          width: 25%;
          padding: 20px;
        }
        
        .row:after {
          content: "";
          display: table;
          clear: both;
        }
        
        @media screen and (max-width: 992px) {
          .column {
            width: 50%;
          }
        }
        
        @media screen and (max-width: 600px) {
          .column {
            width: 100%;
          }
        }

Using media queries in CSS we can:

  • Make content disappear on certain screen size
  • Make them look totally different (size/shape/content/background etc.)
  • Display some content i.e. only on mobile phones
  • Automatically resize media (photo/video) to fill entire space available

Media query syntax

A media query consists of a media type and can contain one or more expressions, which resolve to either true or false. 


@media not|only mediatype and (expressions) {
    CSS-Code;
    }

The result of the query is either true or false which defines whether given style set will be applied by browser or not.

You can define numerous condition in your query like:

  • Targeting different media types - print/ screen or speech synthesizers
  • Targeting media features like min/max width/height, orientation (landscape/portrait)
  • Use logical operators like NOT or AND
  • And many more....
